# Design System Memory

## Product direction
Style: modern, clean, calm, product-focused
Tone: professional, intentional, premium without excess
Goal: make interfaces feel trustworthy, usable, and refined

## Spacing scale
Use this spacing scale consistently:
- 4
- 8
- 12
- 16
- 24
- 32
- 40
- 48

## Typography
### General
- Prioritize readability and hierarchy
- Avoid random text sizes
- Keep typography calm and intentional

### Suggested hierarchy
- Page title
- Section title
- Body text
- Supporting/helper text
- Label text

### Guidelines
- Headings should be clear and strong
- Supporting text should be muted, but still readable
- Avoid weak contrast and washed-out body copy

## Colors
### Foundation
- Use a restrained neutral palette
- Avoid pure black unless necessary
- Avoid overly bright accent usage

### Usage
- One primary accent color
- Neutral surfaces for layout
- Strong readable text contrast
- Use color with purpose, not decoration

## Radius
Use consistent corner rounding:
- Small elements: 8px
- Inputs/buttons: 10px to 12px
- Cards/containers: 14px to 16px

Do not mix many different radius values across one screen.

## Borders
- Use subtle borders
- Avoid thick outlines
- Borders should support structure, not dominate it

## Shadows
- Prefer minimal shadows
- Use soft depth only where it improves separation
- Avoid floating-card overload

## Buttons
### Primary button
- Clear visual priority
- Comfortable horizontal padding
- Consistent height
- Strong but clean hover/focus states

### Secondary button
- Lower emphasis than primary
- Still clearly interactive
- No unnecessary visual noise

## Inputs
- Clear labels above or near input
- Consistent height and padding
- Focus state must be obvious
- Error state must be readable and calm

## Cards and containers
- Prefer fewer, stronger containers
- Avoid unnecessary nesting
- Use spacing inside cards to create hierarchy
- Let content breathe

## Lists and tables
- Prioritize scanning
- Keep spacing even
- Align text and actions consistently
- Use muted separators if needed

## Interaction design
- Hover states should be subtle and deliberate
- Motion should support understanding, not distract
- Focus states must be accessible
- Disabled states should remain understandable

## Accessibility
- Maintain readable contrast
- Ensure buttons and fields are obvious
- Never rely only on color to communicate meaning
- Preserve keyboard/focus usability

## Design principles
- Simplicity over clutter
- Consistency over novelty
- Hierarchy over decoration
- Usability over trendiness
- Calm confidence over flashy UI

## Anti-patterns
Do not generate:
- AI-looking nested card layouts
- random section spacing
- excessive shadows
- thick borders on everything
- too many competing buttons
- noisy dashboards with no visual rhythm
- overly rounded components everywhere
- weak contrast text on tinted surfaces

## Reference feel
Aim for the clarity and restraint often seen in high-quality product interfaces:
- strong spacing rhythm
- clean typography
- minimal but polished surfaces
- focused use of color
- subtle interaction feedback
